About the Project

The Serverless Notes App is a lightweight, secure, and scalable web application for creating, editing, searching, and organizing personal notes. The frontend is a static single-page application hosted on Amazon S3 (served via CloudFront), while the backend is fully serverless: API Gateway routes requests to Lambda functions which implement authentication callbacks and CRUD operations. User identities and tokens are managed with Amazon Cognito and notes are persisted in DynamoDB.

üîç System Architecture

  • Frontend (S3 + CloudFront): Static SPA (HTML/CSS/JS) served from an S3 bucket (notes-app-frontend-varsha) with CloudFront for HTTPS and caching.
  • Authentication: Cognito User Pool manages sign-up, sign-in, password reset, and issues JWTs for authenticated requests.
  • Auth Callback: /auth/callback route handled by Lambda:authorize-cognito-notes-app.
  • API Layer: API Gateway (Notes-Rest-Api) validates tokens (via Cognito) then forwards requests to notes-api Lambda (proxy integration).
  • Data Storage: DynamoDB table NotesTable-App holds notes using composite key (userId, noteId) enabling efficient per-user queries.
  • Typical Request Flow: Browser ‚Üí CloudFront/S3 ‚Üí (login via Cognito) ‚Üí API Gateway ‚Üí Lambda ‚Üí DynamoDB ‚Üí Lambda ‚Üí API Gateway ‚Üí Browser.

üß© UI / Content Types (What the app contains)

  • Authentication Forms: Sign-up, Sign-in, Forgot Password, Confirm Account.
  • Note Editor: Rich-text or Markdown editor with live preview (Title, Body, Tags).
  • Note List / Dashboard: Paginated list of notes, sort by updated_at, search and filter by tags.

  • CRUD Endpoints (API): GET /notes, POST /notes, GET /notes/{id}, PUT /notes/{id}, DELETE /notes/{id}.
  • Client-side Validation: Form validation for title length, required fields, and safe HTML sanitization.
  • Search: Client-side keyword search + optional server-side full-text index via DynamoDB secondary indexes (if enabled).
  • Offline / Sync (optional): LocalStorage caching and background sync to support intermittent connectivity.
  • Settings: Profile (display name), theme (light/dark), export/import notes JSON.
  • Telemetry / Logging: CloudWatch logs for Lambda + basic usage metrics (API latency, errors).
