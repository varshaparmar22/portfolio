Airtable-Based Contractor Management and LLM Evaluation System

About the Project

It involved designing and implementing a robust, serverless data model and automation system using Airtable as the primary database for managing contractor applications. The core goal was to create a structured input flow, automate data management (compression/decompression), implement multi-factor shortlisting rules, and integrate an external Large Language Model (LLM) for automated qualitative application review and enrichment.

Key Features and Architecture

  • 1. Structured Data Model (Airtable Schema) A normalized database schema was established within Airtable, consisting of three linked tables plus two helper tables to capture application details and manage automation5. • Applicants (Parent Table): Stores one row per applicant66. Key fields include: Applicant ID (primary), Compressed JSON, Shortlist Status, LLM Summary, LLM Score, and LLM Follow-Ups77. • Child Tables (Linked): All child tables are linked back to Applicants by Applicant ID8. o Personal Details (One-to-one link to Applicants). o Work Experience (One-to-many link to Applicants). o Salary Preferences (One-to-one link to Applicants). • Helper Table: Shortlisted Leads (Auto-populated when shortlisting rules are met)121212.
  • 2. User Input and Form Flow To simulate a multi-table input flow, three separate Airtable forms were used (one per child table). Applicants are required to submit all three forms, each pre-filling or asking for the Applicant ID to ensure data linkage.
  • 3. Data Compression and Normalization (Python) Two critical local Python scripts were developed to manage data efficiency and integrity: • JSON Compression Script: Gathers data from the three linked tables, builds a single JSON object (e.g., storing personal, experience, and salary details), and writes it to the Compressed JSON field. • JSON Decompression Script: Reads the Compressed JSON and performs an upsert of child-table records to reflect the JSON state, supporting necessary updates and edits to the normalized data.
  • 4. Automated Lead Shortlisting An automation system evaluates applicants against defined, multi-factor criteria after the compression step. If all criteria are met, a record is created in the Shortlisted Leads table. • Criteria: o Experience: $\ge 4$ years total OR worked at a Tier-1 company (e.g., Google, Meta, OpenAI). o Compensation: Preferred Rate $\le \$100$ USD/hour AND Availability $\ge 20$ hrs/week. o Location: In US, Canada, UK, Germany, or India. • Output: Populates the Score Reason field with a human-readable explanation.
  • 5. LLM Evaluation and Enrichment Integration A robust qualitative review step was implemented by calling an external LLM endpoint26262626. • Trigger: After Compressed JSON is written or updated. • Security: Read API key securely from an Airtable Secret or environment variable (not hard-coded). • Prompt Engineering: Feeds the full JSON to the LLM and asks it to perform four specific tasks (Summarize, Rate, Flag Issues, Suggest Follow-ups). • Robustness: Includes validation to log API call failures and implement exponential backoff for up to 3 retries. • Structured Outputs: Writes to LLM Summary ($\le 75$ words), LLM Score (1-10), and LLM Follow-Ups (bullet list) fields on the Applicants table.

Technologies Used
